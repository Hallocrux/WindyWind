## 2. 截至 2026-04-05 的现实基线

当前计划必须以已经落地的主线为前提，而不是从零重开一条新路线。

### 2.1 当前已经稳定的部分

- 表格主线默认入口已经固定在 `main.py -> src/current.pipeline.main`。
- 当前主数据已经切换为 `dataset_manifest.csv + 工况{ID}.csv`。
- 当前默认清洗口径已经固定：
  - 删除首尾连续缺失段；
  - 中间连续缺失 `<=5` 行时插值；
  - 中间连续缺失 `>5` 行时删除并切成连续段；
  - 窗口只在连续段内部生成。
- 当前默认实验口径已经固定为按工况做 `Leave-One-Condition-Out`，不按窗口随机切分。

### 2.2 当前已经拿到的结果

- 当前共有 `20` 个工况，其中 `19` 个带标签，`1` 个无标签。
- 当前主流程共生成 `866` 个窗口样本。
- 当前全局最优模型是 `Ridge + VIB_FT_RPM`，`case_mae = 0.3104`。
- 当前最佳 `rpm-free` 模型是 `Ridge + VIB_FT`，`case_mae = 0.4507`。
- `工况2.csv` 当前只能走 `rpm-free` 回退推理，预测值是 `3.3279 m/s`。

### 2.3 当前已经确认的限制

- 新增 `工况15`、`工况16`、`工况17` 的缺失水平显著高于旧工况。
- 删除首尾连续缺失段之后，边界段仍不能直接视为稳定中段。
- 当前保留下来的窗口中，受重缺失影响的窗口占比已经降到 `0.0000%`，说明主瓶颈不再只是“脏窗口没有被过滤”，而是特征表达和工况差异本身。

## 3. 计划目标

这份计划的目标不是“全面换模型”，而是在当前 `src/current/` 主线上回答四个更现实的问题：

1. 能否先把当前 `rpm-free` 主线做强，而不是继续依赖 `rpm` 作为主要解释来源。
2. 在现有窗口级表格样本口径下，特征工程还能带来多少稳定收益。
3. 当前性能上限里，有多少来自数据质量与窗口策略，而不是模型类型本身。
4. 在当前样本规模下，直接吃原始窗口时序的新模型是否真的值得进入下一阶段。

## 4. 边界与原则

### 4.1 不做的事

- 不在这一轮计划里直接替换 `src/current/` 主线。
- 不把视频 RPM 子项目和表格主线重新混成一个大方案。
- 不在旧特征上大规模刷模型。
- 不把重型深度学习当成本轮的默认主攻方向。

### 4.2 必须保持的原则

- 继续以“工况”为分组单位做评估。
- 继续同时报告 `rpm-free` 和 `rpm-aware` 两类结果，但优先看 `rpm-free`。
- 继续以 case-level 指标作为主指标，而不是只看窗口级误差。
- 所有新探索继续放在 `src/try/`，并配套说明文件与 `outputs/try/` 产物。
- 只有当一条新路线在当前口径下形成了稳定证据，才考虑进入 `src/current/`。

## 5. 分阶段计划

### 5.1 第一阶段：先把当前表格主线做强

这一阶段是整个计划的首要任务，因为它离当前主线最近，也最容易形成可比较的结论。

阶段目标：

- 以当前 `Ridge + VIB_FT` 为 `rpm-free` 基线，建立更强、更稳的 `rpm-free` 主线。
- 判断当前表格路线里，真正有收益的特征增强方向是什么。
- 判断当前 `Ridge` 是否仍然是增强特征后的合理默认模型。

建议推进方向：

- 稳健时域特征增强。
- 更完整的频域形状描述。
- 跨通道结构关系特征。
- 质量感知特征。

阶段要求：

- 所有增强都必须相对当前基线做分组比较。
- 先做成组消融，不做无边界的组合爆炸。
- 第一阶段结束时，至少要能回答“哪一类增强方向值得保留到正式主线”。

阶段产出：

- 一份增强特征对比表。
- 一份按工况的误差变化汇总。
- 一份是否更新 `rpm-free` 默认基线的结论记录。

### 5.2 第二阶段：补数据质量与窗口策略诊断

这一阶段不是另起一条模型线，而是用来解释当前性能上限和难工况来源。

阶段目标：

- 判断当前误差里，有多少是由高缺失工况带来的。
- 判断当前默认 `5s` 窗长是否仍是合适折中。
- 判断边界段问题是否需要继续影响正式训练口径。

建议推进方向：

- 针对 `工况15`、`工况16`、`工况17` 做单独诊断。
- 比较不同缺失筛选强度对结果的影响。
- 在少量候选窗长之间做对比，而不是做大范围窗口搜索。
- 把边界段与中段差异的已有发现纳入解释框架，而不是重复从头讨论。

阶段要求：

- 这一步优先解释“为什么当前结果会这样”，而不是追求新的最好分数。
- 只有当质量或窗长调整带来稳定收益，才考虑修改正式默认口径。

阶段产出：

- 一份困难工况诊断记录。
- 一份缺失筛选影响对比表。
- 一份窗长策略建议。

### 5.3 第三阶段：再评估轻量原始时序模型

这一阶段应建立在前两个阶段完成之后，而不是提前跳过去。

阶段目标：

- 判断原始窗口时域数据中，是否还有当前手工特征没有抓住的信息。
- 判断在当前样本规模下，轻量端到端路线是否真的有继续投入的价值。

建议推进方向：

- 自动时序变换后接轻量回归头的路线。
- 小型时序卷积类路线。
- 只保留少量、代表性强的候选，而不是铺开很多深度模型。

阶段要求：

- 这一阶段的比较对象应是“增强后的 `rpm-free` 主线”，而不是旧基线。
- 只有在按工况留一的口径下持续优于增强后的 `rpm-free` 基线，才值得进入下一轮。
- 如果结果只是个别工况偶然变好，或者方差明显变大，应视为探索性结果，而不是主线候选。

阶段产出：

- 一份轻量原始时序模型对比表。
- 一份是否继续投入端到端路线的阶段判断。

### 5.4 第四阶段：最后再做少量模型扩展

这一阶段的前提是：前面已经有了更强的特征基线，或者已经知道轻量时序模型是否值得继续。

阶段目标：

- 判断增强特征之后，是否开始出现比 `Ridge` 更合适的小样本模型。
- 避免把主要时间花在模型穷举上。

建议推进方向：

- 只保留少量适合小样本、高相关特征场景的候选模型。
- 只在“当前最佳特征组”和“当前最佳特征组 + rpm”上测试。

阶段要求：

- 如果 `Ridge` 仍然最稳，应明确保留它作为默认主线，而不是为了“模型更高级”硬切换。
- 如果新模型只在 `rpm-aware` 情况下收益明显，而 `rpm-free` 仍无改进，结论也应分开记录。

阶段产出：

- 一份小样本模型补充比较表。
- 一份是否替换默认模型的结论记录。

## 6. 推荐执行顺序

按当前仓库进度，推荐顺序如下：

1. 先完成第一阶段，建立更强的 `rpm-free` 表格基线。
2. 再完成第二阶段，解释高缺失工况和窗口策略对结果的影响。
3. 之后再进入第三阶段，测试轻量原始时序模型是否值得继续。
4. 最后再做第四阶段，只补少量模型扩展。

这个顺序的含义是：

- 先提升最贴近当前主线的部分；
- 再解释当前误差来源；
- 再决定是否引入更重的新路线；
- 最后才做模型层面的补充比较。

## 7. 阶段完成后的预期结论

如果这份计划按顺序推进，阶段结束后应至少能明确回答以下问题：

1. 当前 `rpm-free` 主线能否明显优于 `Ridge + VIB_FT`。
2. 当前更值得保留的是哪类特征增强方向。
3. 高缺失工况和边界段问题对误差的影响有多大。
4. 当前 `5s` 窗长是否仍是合理默认值。
5. 轻量原始时序模型是否值得进入下一轮主线候选。
6. 在增强特征之后，`Ridge` 是否仍然应保留为默认模型。

## 8. 一句话总结

`2026-04-05` 这份计划的核心不是“重新发明一条新主线”，而是基于已经跑通的 `src/current/`，先把表格 `rpm-free` 主线做强，再解释质量与窗长约束，最后才审慎评估原始时序模型和值得补测的小样本模型。
